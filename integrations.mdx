## Overview

Tables Guru supports seamless integrations with popular third-party applications, webhooks for custom automations, document embedding in external sites, and data exports to common formats. These features enable you to incorporate Tables Guru into your existing workflows, automate updates, and share content effortlessly.

<Callout kind="info">
  All integrations use secure HTTPS endpoints at `https://api.example.com` and require your API key for authentication.
</Callout>

## Webhook Setup

Set up webhooks to automate responses to events in Tables Guru, such as document updates or new table creations.

<Steps>
  <Step title="Create Webhook" icon="plus">
    Navigate to Settings > Integrations > Webhooks. Click "New Webhook" and enter your endpoint URL, like `https://your-webhook-url.com/webhook`.
  </Step>
  <Step title="Select Events" icon="list">
    Choose events: `document.created`, `table.updated`, or `user.invited`.
  </Step>
  <Step title="Test and Save" icon="check-circle">
    Send a test payload and verify receipt on your server.
  </Step>
</Steps>

Here's an example webhook payload:

<CodeGroup tabs="JSON,JavaScript">
  ```json
  {
    "event": "table.updated",
    "documentId": "doc_123abc",
    "tableId": "tbl_456def",
    "timestamp": "2024-10-15T14:30:00Z",
    "userId": "user_789ghi"
  }
  ```
  ```javascript
  app.post('/webhook', (req, res) => {
    const payload = req.body;
    if (payload.event === 'table.updated') {
      // Handle table update
      console.log(`Table ${payload.tableId} updated in doc ${payload.documentId}`);
    }
    res.status(200).send('OK');
  });
  ```
</CodeGroup>

<ResponseField name="event" field-type="string" required="true">
  The type of event that triggered the webhook.
</ResponseField>

<ResponseField name="documentId" field-type="string">
  ID of the affected document.
</ResponseField>

## Embedding Documents

Embed Tables Guru documents directly into external websites or applications for live, interactive views.

<Expandable title="Iframe Embedding" default-open="true">
  Use this method for simple, secure embeds.

  ```html
  <iframe
    src="https://tablesguru.example.com/embed/doc_123abc?theme=dark"
    width="100%"
    height="600"
    frameborder="0">
  </iframe>
  ```

  Customize with query params: `?theme=light&readOnly=true`.
</Expandable>

<Expandable title="JavaScript SDK">
  For advanced interactivity, load the SDK.

  ```javascript
  import { TablesGuruEmbed } from '@tablesguru/embed-sdk';

  const embed = new TablesGuruEmbed({
    container: '#embed-container',
    documentId: 'doc_123abc',
    apiKey: 'YOUR_API_KEY'
  });
  embed.render();
  ```
</Expandable>

<Callout kind="tip">
  Always use `readOnly=true` for public embeds to prevent unauthorized edits.
</Callout>

## Exporting to Common Formats

Export your tables and documents programmatically or via the UI.

<Tabs>
  <Tab title="CSV Export" icon="file-text">
    ```bash
    curl -H "Authorization: Bearer YOUR_API_KEY" \
         https://api.example.com/v1/documents/doc_123abc/export?format=csv \
         -o data.csv
    ```
  </Tab>
  <Tab title="JSON Export" icon="database">
    ```bash
    curl -H "Authorization: Bearer YOUR_API_KEY" \
         https://api.example.com/v1/tables/tbl_456def/export?format=json \
         -o data.json
    ```
  </Tab>
</Tabs>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token: `Bearer YOUR_API_KEY`.
</ParamField>

<ParamField query="format" param-type="string" required="false">
  Export format: `csv`, `json`, or `pdf`.
</ParamField>
